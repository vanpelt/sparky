Trust model
Privacy, security & sensitive data.
01
PRIVACY
Built on top of GitHub's privacy model.

Sessions tied to an org repo follow that repo's GitHub permissions. Sessions on personal or non-git work stay private to you.

mark any session private disable sharing per-repo disable sharing per-user
02
SECURITY
Short-lived tokens. SSO.

Short-lived tokens in your system keychain, rotated automatically. SSO/SCIM for org access. Infra runs in private VPCs.

keychain + rotation SSO & SCIM VPC + private connect
03
SENSITIVE DATA
Redacted before it leaves your machine.

Secrets are scrubbed on your machine, before anything leaves it.

13 / 20
Five Learnings
01
Rich trajectories are a data flywheel.
Every prompt, tool call, diff — across every agent. A dataset that didn't exist before.
02
Seeing teammates' sessions is legitimately useful.
"How would Tim have done this?" is a one-click question.
03
The leaderboard is loved — and tricky.
Highest-traffic page. Rewards token burn, not outcomes.
04
Live + Overview unlock parallel work streams.
Tail one session while another runs in the background — without losing the thread.
05
Fork is a lifesaver when Anthropic is down.
Hand off a stuck session to another agent in one click — keep moving while the dust settles.

learning · 03
17 / 20
The hard one
The leaderboard is loved.
It also rewards the wrong thing.
V1 · TODAY
token count
Rewards token burn. Ignores caching. Says nothing about whether the code actually shipped.
V2 · NEXT
outcomes
Merged PRs, code survival after 30 days, cost per merged line. Reward what ships.
// incentives shape behavior. if we get the scoreboard wrong, the players optimize for the wrong game.

roadmap · bet 02 · insights v2
22 / 23
02
Bet · 02 · Insights V2
A classifier that earns its keep.
The pipeline that turns trajectories into "here's what's broken." Today it's noisy. Next, it's measured.
V1 · today
Noisy.
  • One-shot classification, no eval rigor.
  • False positives drown the real signals.
  • Surfaces single-session issues, not what's consistently breaking across users.
  • Insights you read — not insights you act on.
V2 · next
Evaluated.
  • Built on Weave evals — measured, not vibes.
  • Patterns that repeat: what's failing for everyone in this repo?
  • Apply-the-fix UX — from insights to PR in one command.
  • Falsifiable. When the fix doesn't work, we see it.
// we're eating Weave's dog food. it's the only way this gets better.
